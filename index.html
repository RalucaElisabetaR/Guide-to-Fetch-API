Comprehensive Guide to Fetch API

1. Introduction to Fetch API

The Fetch API provides a JavaScript interface for accessing and manipulating parts of the HTTP pipeline, such as requests and responses. It's a modern alternative to XMLHttpRequest and is promise-based.

2. Basic Usage

For a simple GET request:

fetch('https://api.example.com/data')
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Error:', error));

3. Making Different Types of Requests

GET Request

Default method used to retrieve data.

POST Request

To submit data to a server:

fetch('https://api.example.com/submit', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({ key: 'value' }),
})
.then(response => response.json())
.then(data => console.log(data))
.catch(error => console.error('Error:', error));

4. Handling Responses

Check the response status and parse the body accordingly.

5. Error Handling

Handle errors by checking the response's ok property.

6. Advanced Usage

Headers

Add custom headers to your request:

fetch('https://api.example.com/data', {
  headers: new Headers({
    'Authorization': 'Bearer your_token_here',
  }),
})

CORS

Understand server-side Access-Control-Allow-Origin headers for cross-origin requests.

Caching

Control caching behavior of your requests with the cache option.

7. Practical Examples

Fetching a list of posts and logging them:

fetch('https://jsonplaceholder.typicode.com/posts')
.then(response => response.json())
.then(json => console.log(json))
.catch(error => console.error('Unable to fetch posts:', error));
